EFCore - 相同的LINQ被翻譯為不同的SQL語句

2023-06-05
efcore

  • 相同的LINQ被翻譯為不同的SQL語句

  • 1 - 下列三條在不同資料庫系統的語句，都具有相同的語義

  • (1) Sql Server - select top(3) * from table

  • (2) MySQL - select * from table limit 3

  • (3) Orable - select * from table where rownum <= 3

  • 2 - 有鑑於1所述，一條使用了EF Core的C#語句會被針對不同資料庫系統設計的EF Core Provider翻譯成可能具有細微差別的不同SQL語句

  • EF Core的遷移腳本大部分都不能跨資料庫系統使用，例如:EF Core的Sql Server遷移腳本可能不能在MySql使用

  • 透過Add-Migration命令並添加-OutputDir參數，可在同一個專案中針對不同資料庫系統生成不同目錄下的不同遷移腳本

  • 試查看下列C#語句被翻譯為何種SQL語句(透過簡單日誌查看)
  • 翻譯為Sql Server中的SQL語句

  • 安裝EF Core MySQL Provider並添加下列程式碼

  • NuGet: Pomelo.EntityFrameworkCore.MySql -Version 7.0.0

  • 若當前的EF Core遷移腳本仍然為Sql Server遷移腳本並使用Sql Server遷移腳本Update-Database至Mysql，會出現如下錯誤

  • 由上述可得知，EF Core的遷移腳本大部分都不能跨資料庫系統使用(有些內容會遷移成功，但絕大部分內容會遷移失敗)，此時必須透過Add-Migration命令並添加-OutputDir參數，在同一個專案中針對不同資料庫系統生成不同目錄下的不同遷移腳本(在此不示範)

  • 因無法使用Sql Server遷移腳本Update-Database至MySQL，故先將Sql Server遷移腳本刪除，再透過Add-Migration生成MySQL遷移腳本，再Update-Database至MySQL，就可繼續進行SQL語句的翻譯工作

  • 翻譯為MySQL中的SQL語句

  • 安裝EF Core PostgreSQL Provider並添加下列程式碼

  • NuGet: Npgsql.EntityFrameworkCore.PostgreSQL -Version 7.0.4

  • 若當前的EF Core遷移腳本仍然為MySQL遷移腳本並使用MySQL遷移腳本Update-Database至PostgreSQL，會出現如下錯誤

  • 由上述又可得知，EF Core的遷移腳本大部分都不能跨資料庫系統使用(有些內容會遷移成功，但絕大部分內容會遷移失敗)，此時必須透過Add-Migration命令並添加-OutputDir參數，在同一個專案中針對不同資料庫系統生成不同目錄下的不同遷移腳本(在此不示範)

  • 因無法使用MySQL遷移腳本Update-Database至PostgreSQL，故先將MySQL遷移腳本刪除，再透過Add-Migration生成PostgreSQL遷移腳本，再Update-Database至PostgreSQL，就可繼續進行SQL語句的翻譯工作

  • 翻譯為PostgreSQL中的SQL語句
