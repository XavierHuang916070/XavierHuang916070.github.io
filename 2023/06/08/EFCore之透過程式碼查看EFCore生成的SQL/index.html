EFCore - 透過程式碼查看EFCore生成的SQL

2023-06-08
efcore

  • 透過程式碼查看EFCore生成的SQL

  • 不是已經有SQL Server Profiler可以查看EF Core生成的SQL了?為何還要透過程式碼查看EF Core生成的SQL? 因為SQL Server Profiler是查看SQL Server伺服器上所有的活動，因此顯得較為雜亂無章、較難查看，若SQL Server伺服器是供多人開發使用，則使用SQL Server Profiler將會更難查看，且SQL Server Profiler只能用於SQL Server，不支援其他資料庫系統

  • 1- 標準日誌

  • 2 - 簡單日誌

  • 3 - ToQueryString

  • 標準日誌查看法、簡單日誌查看法均無法直接看清某個操作所對應到的SQL語句，且在操作過多時非常容易混亂

  • DbSet實現了IQueryable介面，EF Core的Where方法為IQueryable介面的擴展方法且其返回值類型為IQueryable類型，除此之外，IQueryable介面還具有擴展方法ToQuetyString方法可獲得對應的SQL語句

  • 查詢操作時，不需要真正執行查詢就可獲得對應的SQL語句

  • 僅限用於查詢操作

  • 總結

  • 1 - 撰寫測試性程式碼時，建議使用簡單日誌

  • 2 - 需要正式記錄SQL語句給維運人員、審核人員排查故障時，建議使用標準日誌

  • 3 - 需要從複雜的查詢操作中立即看到對應的SQL語句時，建議使用ToQueryString
