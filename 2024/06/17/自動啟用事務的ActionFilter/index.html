自動啟用事務的Action Filter

2024-06-17
aspdotnetcore

  • 自動啟用事務的Action Filter

  • 案例需求 - 自動啟用、提交、回滾事務(又稱之為交易、Transaction)

  • Ps: 資料庫事務: 資料庫操作不是全部成功，就是全部失敗

  • Ps:若將一段使用了EF Core進行資料庫操作的程式碼放到TransactionScope所宣告的範圍時，此段程式碼會自動被標記為事務

  • Ps: TransacrionScope實現了IDisposable介面，若一個TransactionScope的Object在最後自動呼叫Dispose()之前沒有手動呼叫Complete()，則事務無論如何都會被回滾

  • Ps: TransactionScope()支持嵌套式事務

  • Ps: .Net Framework中的TransactionScope在進行跨資料庫實例之操作時，會被視為分布式事務，此時會啟用MSDTC機制來協調跨資料庫實例之操作，因此，不同資料庫實例之操作，會保證一段時間內的資料一致性

  • Ps: .Net Core、.Net中的TransactionScope在進行跨資料庫實例之操作時，不會被視為分布式事務，而是會被視為最終一致性事務，因此，不同資料庫實例之操作，不會保證一段時間內的數據一致性，只要最終達到數據一致即可
