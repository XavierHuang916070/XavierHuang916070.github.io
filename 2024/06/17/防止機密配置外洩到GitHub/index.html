防止機密配置外洩到GitHub

2024-06-17
aspdotnetcore

  • 防止機密配置外洩到GitHub

  • User Secrets

  • 1 - 將不方便放在appsettings.json文件中的機密配置，改放在專案的secrets.json文件中

  • 2 - 在Asp.Net Core專案上點擊鼠標右鍵，選擇”管理使用者密碼”

  • 3 - secrets.json文件並非放在專案根目錄中，而是放在本地用戶端磁碟上的某路徑，因此專案上傳到GitHub後，機密配置不會外洩到GitHub，如下所示:

  • 4 - 讀取secrets.json Example1
  • 5 - 讀取secrets.json Example2

  • Ps: Ps: User Secrets只適合用於開發環境供開發人員使用，不適合用於生產環境

  • Ps: User Secrets中的機密配置仍然是明文，若不想讓別人直接看到機密配置之明文訊息，請使用中心化配置伺服器來存放機密配置(例如: 採用Azure Key Vault(雲平台配置提供者) )，但中心化配置伺服器仍然無法完全確保機密配置的安全，還必須進一步配合安全防護、監控機制才可更有效確保機密配置的安全

  • Ps: 若因為重裝系統等原因而造成專案於開發環境中的User Secrets需重建，則需重新進行配置，較為麻煩，此為User Secrets的缺點，可改用中心化配置伺服器來解決此問題
