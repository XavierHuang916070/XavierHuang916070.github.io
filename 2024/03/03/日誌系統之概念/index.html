日誌系統 - 概念

2024-03-03
csharpdotnet

  • 日誌、日誌系統

  • 1 - 應用程序、應用系統在運行時生成的重要訊息稱之為日誌，日誌包含了: 事件訊息、Debug訊息、錯誤訊息、警告訊息、故障訊息、性能指標、用戶活動 /用戶操作的常規訊息等，用來管理、監控日誌的系統稱之為日誌系統

  • 2 - 當應用程序、應用系統發生問題時，透過日誌系統可高效查找發生問題的原因

  • .Net Core的日誌級別 - .Net Core的日誌級別共有6種，級別由低到高分別為Trace => Debug => Information => Warning => Error => Critical

  • 日誌提供者(LoggingProvider) - 負責將日誌輸出、推送到控制台、文件、資料庫、接收者郵箱等

  • .Net Core的日誌系統非常靈活，對於業務邏輯程式碼而言，只要注入日誌物件來記錄日誌即可，至於哪些日誌輸出到哪裡、日誌的格式為何、是否輸出等都是在框架部分來進行配置而非在業務邏輯程式碼處進行配置

  • 將日誌輸出到控制台

  • 1 - Nuget: Microsoft.Extensions.Logging、Microsoft.Extensions.Logging.Console

  • 2 - 註冊服務: services.AddLogging(logBuilder=>{logBuilder.AddConsole()}); 需要記錄日誌的業務邏輯程式碼部分，注入ILogger< T >的實例物件即可，T一般就用當前類，這個類名會輸出到日誌，方便定位錯誤

  • 3 - 呼叫LogInformation()、LogError()等方法輸出不同級別的日誌，也支持輸出異常物件

  • Windows EventLog(事件檢視器) - 在Windows平台上部屬的程序、系統運行發生問題時，不妨先去EventLog看看，至於如何將日誌寫入Windows EventLog? 如下所示:
